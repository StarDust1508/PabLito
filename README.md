# PabLito 🇦🇷

Твой личный собеседник, который учит тебя **аргентинскому испанскому** (риоплатский: *vos*, «sh», лунфардо) — с настоящим характером, настроением и памятью. Цель — заговорить и уехать в Аргентину.

Паблито — весёлый **портеньо** из Буэнос-Айреса: шутит, болтает, мягко поправляет по ходу и объясняет максимально по-испански, откатываясь на русский только когда ты реально застрял.

---

## Что уже внутри

- **Живое настроение v2 (главная фича).** Быстрые оси момента (энергия/теплота/терпение) + медленная ось **близости (bond)**, которая копится неделями, почти не дрейфует и имеет пол (не рушится при паузах). Мягкое насыщение у краёв и потолок дельты за ход — без «телепортов» настроения. «Тонус дня» даёт мягкую непредсказуемость. Эмоции определяет сама модель (сигнал в `[[MEM]].mood`), регексы — фолбэк. Дуга отношений: знакомство → приятели → близкие → давние друзья, что открывает более тёплое поведение. → `src/core/mood.ts`
- **Эмоциональная память.** Паблито запоминает значимые моменты (трудный разговор, победа) и к месту вспоминает их — раздел «Наши моменты» в экране 🧠. → `src/core/memory.ts`
- **Этические предохранители.** Никакой вины за паузы, дефицита и токсичной зависимости; не подменяет живое общение; на максимуме близости не даёт новых прав на давление. Прописано в персоне и в движке. → `src/core/personality.ts`, `docs/design/DECISIONS.md`
- **Долгая память v2.** Типизированные факты (личность/цели/факты/предпочтения/моменты/обещания) с заменой устаревших: «переехал» аннулирует старый адрес, а не плодит дубли. Дедуп по нормализованному тексту, закрепление важного (имя, цель), выборка по бюджету символов. → `src/core/memory.ts`, `src/core/memory-logic.ts`
- **Прозрачная память.** Экран «Обо мне» (🧠): видишь и правишь/удаляешь всё, что Паблито о тебе знает, или стираешь целиком. → `src/screens/MemoryModal.tsx`
- **Транскрипт переживает перезапуск.** Диалог пишется в SQLite; при возврате в течение 30 мин сессия продолжается, при уходе в фон Паблито делает краткое резюме на следующий раз.
- **SM-2 повторение.** Слова закрепляются по ease-factor; «использовал верно вживую» двигает вперёд, трудные возвращаются чаще. → `src/core/memory-logic.ts`
- **Голос.** Микрофон → распознавание `whisper-1`; ответы озвучиваются (TTS NavyAI, откат на системный `es-AR`). → `src/core/voice.ts`
- **Характер и педагогика.** Персона + мягкие исправления + погружение собраны в системный промпт. → `src/core/personality.ts`
- **Стриминг ответов.** Паблито «печатает» ответ по словам (SSE через `expo/fetch`), с откатом на обычный запрос. → `src/api/navy.ts`, `src/hooks/usePablito.ts`
- **Оценка произношения.** Read-aloud: фраза → читаешь вслух → whisper слышит → пословная подсветка (верно/почти/мимо), счёт с частичными баллами и подсказки по типичным ошибкам русскоязычных (r/rr, b/v, финальная s) + риоплатские («ll/y → ш»). → `src/core/pronunciation.ts`, `src/screens/PronunciationModal.tsx`
- **Три города на главном экране.** Саратов, Буэнос-Айрес, Москва: местное время, погода, влажность, «душно/жарко» (Open-Meteo, без ключа). → `src/api/weather.ts`, `src/components/CityStrip.tsx`
- **Онбординг-разговор.** При первом запуске Паблито знакомится, узнаёт имя и дату переезда прямо в диалоге (без анкет), сохраняет в профиль через `[[SET]]`. → `src/core/personality.ts`
- **Прогресс и мотивация.** Стрик 🔥 (не обнуляется в 0), счётчик дней до Аргентины рядом с вехами, экран «выученные слова» из SRS, дневной переключатель Урок/Болталка. → `src/core/progress.ts`, `src/screens/LearnedWordsModal.tsx`
- **Прокси для ключа (опционально).** Cloudflare Worker прячет ключ NavyAI на сервере. → `proxy/`
- **Интерфейс в стиле malvah.** Редакторский минимализм, светлая/тёмная тема, моно-лейблы. → `src/theme/theme.ts`, `src/screens/ChatScreen.tsx`

## Структура

```
App.tsx / index.ts        точка входа
src/config.ts             конфиг + твой профиль из интервью

```

## Запуск (разработка)

```bash
npm install
cp .env.example .env      # вставь свой ключ в .env
npx expo start            # открой в Expo Go на телефоне (QR)
```

> Голос и SQLite в Expo Go работают. Если что-то из нативного заупрямится — используй dev build (`npx expo run:android`).

## Сборка APK для Samsung S25

Тут в песочнице нет Android-тулчейна, поэтому APK собирается **на твоей стороне**. Самый простой путь — облако EAS (бесплатно):

```bash
npm install -g eas-cli
eas login                                   # заведи аккаунт expo.dev, если нет
eas build:configure
eas build -p android --profile preview      # соберёт APK в облаке и даст ссылку
```

По ссылке скачиваешь `.apk` прямо на Samsung и ставишь (разреши «установку из неизвестных источников»).

**Важно про ключ при облачной сборке:** переменные `EXPO_PUBLIC_*` вшиваются в билд. Чтобы ключ не лежал в репозитории, задай их как секреты EAS:

```bash
eas secret:create --name EXPO_PUBLIC_NAVY_API_KEY --value "sk-navy-..." --scope project
```

Альтернатива без облака: `npx expo run:android` на компьютере с Android Studio — соберёт и поставит на подключённый телефон.

## 🔐 Безопасность (прочитай)

1. **Перевыпусти ключ.** Тот ключ, что ты присылал в переписку, считай засвеченным — сделай новый в кабинете NavyAI и держи его только в `.env` (он уже в `.gitignore`).
2. **Ключ в приложении можно вытащить.** Любой `EXPO_PUBLIC_*` виден в собранном APK при декомпиляции. Для приложения «только для себя» это ок.
3. **Хочешь спрятать ключ полностью — подними мини-прокси.** Разверни бесплатный Cloudflare Worker, который хранит ключ у себя и просто пересылает запросы на `api.navy`. Тогда в `.env` укажи URL воркера вместо `https://api.navy/v1`, а сам ключ в приложение не попадёт вовсе. Скажи мне — сделаю воркер отдельной задачей.

## Заметки по api.navy

Клиент написан под OpenAI-совместимый формат: `/chat/completions`, `/audio/transcriptions`, `/audio/speech`. Если у NavyAI какой-то путь или имя голоса TTS отличается — правится только в `src/api/navy.ts`. Модели заданы в `.env`: чат `deepseek-v4-flash`, разборы `deepseek-v4-pro`, распознавание `whisper-1`.
