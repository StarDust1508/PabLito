# ТЗ-03: Раздельные чаты по режимам, чистый ввод, светлая тема, продуктовая полировка

Обратная связь с устройства после ТЗ-02. Функции работают, но UX/продуктовый вид требует доводки. Ниже — точечно.
Формат: **проблема → надо → критерий → где**.

---

## 1. 🔴 ГЛАВНОЕ: режимы = ДВА ОТДЕЛЬНЫХ ЧАТА (чиним наслаивание)
- **Проблема:** сейчас Друг и Урок — один и тот же чат с разным системным промптом. При переключении на «Урок» сообщения «Друга» не уходят — треды **наслаиваются друг на друга**. А переключатель режима в шторке меняет тему прямо в фоновом чате (пользователь принял его за сортировку истории).
- **Надо:** Друг и Урок — **два независимых чат-треда**, у каждого своя история. Переключение = показать другой тред целиком (НЕ подмешивать сообщения).
- **Архитектура:**
  - На каждый режим — своя «активная» сессия (`sessions.mode`). У `usePablito` история/лента ключуются по режиму.
  - Переключение режима: сохранить текущий тред → загрузить транскрипт другого режима (или начать новый, если пуст) → **полностью заменить ленту**. Resume/приветствие — по режиму.
  - Настроение/память/профиль — общие (это один Паблито), разделяются только треды разговора.
- **Индикатор на чате:** полупрозрачная **таблетка** «💬 Друг» / «📚 Урок» вверху ленты — всегда видно, в каком треде. Тап по таблетке или по верхнему островку — переключить тред.
- **Убрать переключатель режима из ШТОРКИ.** В шторке остаётся только **история** (просмотр прошлых чатов с бело/синими метками), это не живой переключатель.
- **Критерий:** переключил на «Урок» → видишь урок-тред (свой, пустой при первом разе), друг-чат не подмешан; вернулся на «Друг» → его история на месте; таблетка показывает текущий режим; в шторке НЕТ переключателя режима, только история.
- **Где:** `usePablito.ts` (тред по режиму, загрузка/сохранение по `mode`), `ChatScreen.tsx` (таблетка-индикатор), `ProfileDrawer.tsx` (убрать переключатель режима).

## 2. Убрать слово «РЕЖИМ», чистый островок
- **Надо:** в верхнем баре убрать лейбл «РЕЖИМ». Оставить аккуратный сегмент-островок **«💬 Друг | 📚 Урок»** (он же переключатель тредов).
- Когда верх скрыт (письмо/раскрытие) — маленькая полупрозрачная таблетка режима (из §1) остаётся видимой сверху ленты.
- **Критерий:** над чатом чисто, без слова «РЕЖИМ»; понятно, где переключать.
- **Где:** `ChatScreen.tsx` — `modeBar`.

## 3. Ввод: скрепка + микрофон в отдельном маленьком полупрозрачном островке
- **Проблема:** сейчас иконки разбросаны по строке ввода.
- **Надо:** строка ввода = **[малый полупрозрачный островок: 📎 + 🎙]** · **[островок текста]** · **[кнопка отправки]**. Скрепка и микрофон сгруппированы в один компактный полупрозрачный островок на линии сообщения. Иконки — lucide.
- **Критерий:** attach и voice — в своём аккуратном полупрозрачном островке слева от поля; поле и отправка — как есть (ровные).
- **Где:** `ChatScreen.tsx` — `inputBar`.

## 4. Светлая тема + переключатель темы
- **Проблема:** приложение берёт системную тему (телефон тёмный → тёмный), а хочется светлую.
- **Надо:**
  - Добавить выбор темы в настройках шторки: **Светлая / Тёмная / Системная** (сохранять выбор).
  - Довести **светлую** тему до «дорогого» malvah-вида: тёплый бумажный фон (`#EDEAE3`/`#F5F3EE`), почти чёрный текст (`#141210`), celeste-акцент (`#3E6DDb`), тонкие линии, мягкие тени вместо жирных рамок.
- **Критерий:** можно включить светлую тему; выглядит цельно и премиально (см. макет).
- **Где:** новый флаг темы (kv в `memory.ts` или контекст), применение в `theme.ts`/экранах; тумблер в `ProfileDrawer.tsx`.

## 5. Продуктовая полировка (почему «выглядит плохо» — конкретно)
Не «сделай красиво», а точечно:
- **Пузыри:** убрать моно-лейблы `VOS`/`PABLITO` над каждым сообщением (визуальный шум). Реплики Паблито — с маленьким аватаром слева; выравнивание user справа. Мягкая тень вместо жирной рамки, радиусы консистентные.
- **Верхние плашки** (стрик/счётчик) — компактнее, в одну лёгкую строку, меньше визуального веса.
- **Карточки городов** — облегчить (тоньше, больше воздуха); в развёрнутом чате скрыты.
- **Единая сетка 8pt**, больше воздуха вокруг островов; консистентные радиусы/тени/линии по всему экрану.
- **Кинетический логотип** — приглушить амплитуду, чтобы не спорил с контентом (лёгкое «дыхание», не пёстрый хаос).
- **Иконки** — только lucide, одинаковая толщина/размер.
- **Критерий:** экран читается как «дорогой продукт», а не набор плашек; ничего не «кричит».

- [ ] Над чатом нет слова «РЕЖИМ»; сегмент Друг/Урок чистый.
- [ ] Скрепка+микрофон — в отдельном полупрозрачном островке на линии ввода.
- [ ] В настройках можно выбрать Светлая/Тёмная/Системная; светлая выглядит премиально.
