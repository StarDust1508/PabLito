## 1. Технический фундамент (ставим один раз, иначе сломается сборка)

```bash
npx expo install react-native-reanimated react-native-gesture-handler react-native-svg expo-font expo-image-picker expo-haptics @gorhom/bottom-sheet

```
- `react-native-reanimated` — анимации логотипа/островов на UI-потоке.
- `react-native-gesture-handler` — свайпы (раскрытие, шторка).

### 1.2 Обязательная настройка (частые грабли — прописать точно)
1. **babel.config.js** — плагин reanimated должен быть **последним** в списке:
```js
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ['babel-preset-expo'],
    plugins: [
      ['module-resolver', { root: ['./'], alias: { '@': './src' } }],
      'react-native-reanimated/plugin', // ← ВСЕГДА последним
    ],
  };
};
```
2. **index.ts** — первой строкой импорт gesture-handler:
```ts
import 'react-native-gesture-handler';
import { registerRootComponent } from 'expo';
import App from './App';
registerRootComponent(App);
```
3. **App.tsx** — обернуть всё в `GestureHandlerRootView`:
```tsx
import { GestureHandlerRootView } from 'react-native-gesture-handler';
// <GestureHandlerRootView style={{ flex: 1 }}><SafeAreaProvider>…</SafeAreaProvider></GestureHandlerRootView>
```
4. **app.json** — добавить разрешения/плагины:
```json

"android": { "softwareKeyboardLayoutMode": "pan" }
```
5. **CI:** всё это Expo-совместимо; workflow `npm install` подхватит. New Architecture у нас выключен — reanimated/gesture-handler собираются штатно. Ничего в workflow менять не нужно.

### 1.3 Шрифты (malvah-вайб, открытые лицензии)
Положить `.ttf` в `assets/fonts/`, грузить через `expo-font`:
- **Fraunces** (современный serif, «дорогой» акцент),

## 2. Перевод (доработать до полного)
- **Тап по реплике Паблито** → перевод целиком (уже есть).
- **Двойной тап (или долгий тап) по СЛОВУ** в любой реплике → поповер: слово, перевод, кнопка **«＋ в словарь»** (добавляет в 📖/SRS).
- **Реализация слов:** текст сообщения разбить на токены-слова, каждое — обёрнуто в `Pressable`; двойной тап ловим по времени (2 нажатия < 300 мс) или `onLongPress`. Пунктуацию сохранять, переводить только слово.
- **Кэш:** переводы реплик и слов кэшировать (в состоянии/`Map`), не дёргать модель повторно.
- **Добавление в словарь:** `mem.upsertVocab([{ word, translation, context: 'new' }])`.
- **Критерий:** тап — перевод фразы; двойной/долгий тап по слову — перевод + добавление в словарь; повторный перевод не запрашивается.
- **Где:** `src/screens/ChatScreen.tsx` (`Bubble` → токенизация + поповер), `src/api/navy.ts` (`translate` есть; при желании `translateWord`).

## 3. Чат как ядро: острова и раскрытие

### 3.1 Лента
- Перевести список на **инвертированный** `FlatList inverted` (или `@shopify/flash-list`) — надёжно держит низ при новых сообщениях и подъёме клавиатуры.
- **Критерий:** при письме и новых сообщениях лента всегда прижата к низу, скролл не «прыгает».

### 3.2 Раскрытие чата — две механики
- **Механика 1 — кнопка.** В области чата, верхний угол — полупрозрачная плашка **«Expandir»**. Тап → верхняя зона (города/режим/панель) уезжает, чат на весь экран; кнопка меняется на **«Contraer»**. (Текущее авто-скрытие по фокусу оставить как поведение по умолчанию, кнопка — явное управление.)
- **Механика 2 — свайп острова.** Свайп по **ввод-острову** влево/вправо разворачивает/сворачивает (реверсивно) — через `PanGesture` (gesture-handler) + анимация высоты (reanimated).
- При подъёме клавиатуры верхняя зона скрывается автоматически (уже есть).
- **Критерий:** развернуть/свернуть и кнопкой, и свайпом; плавно (60 fps); состояние стабильно.

### 3.3 Шапка-остров (в развёрнутом режиме)
Аккуратный скруглённый «остров» над лентой, не во всю ширину, лёгкая тень/воздух:
- **слева** — кружок-аватар Паблито;
- **центр** — имя + presence («в сети/печатает…/был недавно»);
- **справа** — две **векторные** иконки (lucide): **`Paperclip`** (§4 фото), **`Search`** (§5 поиск).
- **Критерий:** читается как шапка диалога мессенджера; иконки чёткие, одной системы.

### 3.4 Ввод-остров
- Один скруглённый остров, **длиннее** шапки-острова; текст по центральной оси (готово).
- **слева** — кружок аудио (`Mic`), **справа** — `Send` (обе — lucide, не эмодзи).
- Растёт по высоте до 4–5 строк плавно.
- Микро-анимация нажатия (scale/opacity) + `expo-haptics` на отправку.

### 3.5 Премиальность
Все иконки (Paperclip, Search, Mic, Send, а также 📖→`BookOpen`, 🧠→`Brain`, 🎙 Práctica→`Mic`/`Sparkles`) — из **lucide-react-native**, единая толщина линий, скругления. Заменить эмодзи-заглушки.

## 4. Фото (скрепка) — Паблито разбирает по-испански
- Тап `Paperclip` → выбрать фото (`expo-image-picker`, разрешение из app.json).
- Фото красиво показывается в ленте (в пузыре пользователя).
- **Паблито комментирует фото по-испански** (описывает, задаёт вопрос, вплетает лексику) — сильная практика.
- **Vision-запрос (OpenAI-совместимый формат):**
```jsonc
messages: [{ role: 'user', content: [
  { type: 'text', text: 'Mirá esta foto y comentala en español rioplatense, con onda; preguntame algo sobre ella.' },
  { type: 'image_url', image_url: { url: 'data:image/jpeg;base64,<...>' } }
]}]
```
- **ЗАДАЧА-зависимость:** уточнить у NavyAI имя **vision-совместимой** модели (deepseek-vl / gpt-4o-mini / и т.п.) и её лимиты. До подтверждения — код готовим, модель ставим из `.env` (`EXPO_PUBLIC_MODEL_VISION`).
- Сжимать изображение перед отправкой (`expo-image-picker` `quality`, ресайз) — экономия трафика/токенов.
- **Критерий:** отправил фото → оно в ленте → Паблито отвечает про него по-испански.
- **Где:** `src/api/navy.ts` (`chatVision(imageBase64, prompt)`), `ChatScreen.tsx` (скрепка + пузырь с картинкой), `.env` (`EXPO_PUBLIC_MODEL_VISION`).

## 6. Профиль / настройки — выдвижная шторка
- **Жест:** долгий тап по области аватара + потянуть вправо → выезжает боковая шторка (как в ChatGPT). Технически: `LongPressGesture` → активирует `PanGesture` (иначе конфликт со свайпом раскрытия, см. §9). Плюс запасной вход: тап по аватару.
- **Содержимое:**
  1. **Профиль:** аватар, имя, уровень, дата переезда + счётчик, стрик 🔥. (bond и тон дня НЕ показываем.)
  2. **История чатов — 30 дней:** список сессий, каждая — краткое резюме + **метка режима**: **белая точка = Друг**, **синяя точка = Урок**. Тап → открыть/продолжить сессию.
  3. **Смена режима** Друг/Урок (дубль).
  4. **Настройки:** тема (свет/тьма/системная), звук озвучки вкл/выкл, тумблер «Паблито пишет первым», ссылки на 🧠 «Обо мне» и «стереть память».
- **Данные:**
  - Добавить сессии поле **`mode`** (`chat`|`lesson`), проставлять в `startSession()` из текущего режима.
  - `getSessions(sinceDays = 30)` → `[{ id, started_at, summary, mode }]`.
  - **Ретеншн 30 дней:** при старте удалять `messages`/`sessions` старше 30 дней (закрывает и замечание код-ревью про рост БД).
- **Критерий:** лонгпресс+свайп вправо открывает шторку; история 30 дней с бело/синими метками; переключение режима и настройки работают; свайп/тап по затемнению — закрыть.
- **Где:** новый `src/screens/ProfileDrawer.tsx`; `memory.ts` (`sessions.mode`, `getSessions`, `searchMessages`, ретеншн); жест в шапке `ChatScreen.tsx`.

## 7. Кинетический логотип «PabLito»
Везде, где встречается имя (шапка, шапка-остров, splash), каждая буква ритмично меняет облик.

### 7.1 Модель варианта
```ts
type LetterVariant = {
  fontFamily: string;      // из пула (гласные ≠ согласные)
  fontSize: number;        // 22..40
  fontWeight: '400'|'600'|'800'|'900';
  color: string;           // из палитры темы + акцент
  letterSpacing: number;   // -1..3
  rotateDeg: number;       // -8..8
  translateY: number;      // -4..4 (baseline shift)
};
```

### 7.2 Пулы и генерация 40 вариантов на букву
- Два пула: **VOWELS** (a,i,o) и **CONSONANTS** (P,b,L,t) — **разные наборы шрифтов и трансформов**, чтобы ритм читался (например, гласные — Fraunces/serif с наклоном, согласные — гротеск/моно, прямые).
- Для каждого пула детерминированно сгенерировать **40 вариантов** комбинаторно из наборов (fonts × sizes × weights × colors × spacing × rotate × translateY), с фиксированным сидом → стабильный набор.

### 7.3 Анимация и ограничения
- Буквы циклически переключают вариант, **с фазовым сдвигом** между соседями (эффект волны), период ~0.6–1.2 c. Реализация: Reanimated shared value + `useFrameCallback` или `withRepeat(withTiming)` по индексу варианта на каждую букву.
- **Ограничение:** в один момент времени на строке **не более 2 одинаковых вариантов** — планировщик выбора кадра следит за уникальностью индексов среди одновременно показываемых букв.
- **Тап (1 раз):** заморозить — оставить каждую букву в текущем варианте той секунды; повторный тап — снова анимировать.
- **Reduce Motion (доступность):** если включено — статичный красивый вариант, без анимации.
- **Производительность:** только UI-поток (Reanimated), без `setState` на кадр; на слабом устройстве не лагает.
- **Критерий:** имя «дышит» шрифтами/цветами; ≤2 одинаковых буквы в строке; гласные визуально отличаются от согласных; тап замораживает кадр; нет лагов.
- **Где:** новый `src/components/KineticWordmark.tsx`, `assets/fonts/`, загрузка в `App.tsx`; заменить статический `PabLito_` в шапке.

## 9. Дополнения и подводные камни (учесть обязательно)
1. **Конфликт жестов:** свайп раскрытия (по ввод-острову) vs шторка (лонгпресс на аватаре + drag) vs системный «назад». Развести зоны и использовать `Gesture.Exclusive`/`Simultaneous` из gesture-handler; шторку активировать только через лонгпресс.
2. **Клавиатура (Android):** `softwareKeyboardLayoutMode: "pan"` в app.json + инвертированная лента (§3.1) — убирает дёрганье.
3. **Доступность:** `accessibilityLabel` на иконки; логотип уважает Reduce Motion.
4. **Оффлайн:** нет сети → перевод/фото-разбор недоступны — мягкая подсказка, без крэша.
5. **Приватность фото:** изображения только в NavyAI/твой прокси, никуда больше.
6. **Кэш переводов** — не гонять один и тот же текст повторно.
7. **Splash до шрифтов:** не мигать системным шрифтом — держать splash, пока `useFonts` не готов.
8. **Ретеншн = история 30 дней** — одна реализация закрывает и приватность, и рост БД.
