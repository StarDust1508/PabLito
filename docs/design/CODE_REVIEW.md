# PabLito — код-ревью после v2 (память + настроение)

Три субагента проаудировали код по подсистемам. Ниже — что нашли и что сделано.

## ✅ Исправлено сразу

**Блокеры сборки/старта**
- 🔴 Алиас `@/…` не резолвился Metro (пустой babel) → добавлен `babel-plugin-module-resolver` + алиас. Без этого приложение вообще не собиралось.
- 🔴 Гонка в `db()`: параллельные первые вызовы запускали `migrate()` дважды (DDL/RENAME) → кэшируем ПРОМИС БД, а не значение.
- 🔴 Двойная инициализация сессии/приветствия в React StrictMode → guard `inited`.

**Утечки и данные**
- 🔴 Незакрытый `[[SET]]`/`[[MEM]]` при обрыве стрима мог утечь в чат и в TTS → обрезка в `extractProfile` + финальная страховка перед показом/озвучкой.
- 🟠 Гонка двойного `send()` (быстрый двойной тап) → синхронный guard `sending`.
- 🟠 `upsertFact` молча терял не-версионируемый факт при совпадении темы → накопительные виды теперь всегда по ключу-с-текстом.
- 🟠 `clearKnowledge` не чистил транскрипт → модель могла восстановить «стёртые» факты из истории; теперь чистит и `messages`.
- 🟠 Resume мерялся от `started_at` → долгие сессии после force-quit теряли контекст; теперь по последней активности + «осиротевшая» сессия закрывается с авто-резюме.
- 🟠 `resurfaced` моментов рос при каждом холодном старте → инкремент только для новой сессии (`getMomentsPreview` для resume).

**Прочность**
- 🟠 Нет таймаута у стриминга (зависшая сеть → вечный `busy`) → `AbortController` на 45 c.
- 🟠 `expo-av` запись — синглтон без защиты (краш «Only one Recording») → остановка предыдущей перед новой.
- 🟠 `onboarded:true` от модели без реального имени закрывал онбординг → валидация: принимаем только если имя известно.
- 🟠 Промпт мог разрастаться (recap/моменты без лимита) → обрезка по бюджету символов.
- 🟡 Дозирование новых слов в `getDueVocab` (резерв слотов), календарная валидация даты, автоскролл во время стриминга, `exportMemory` включает моменты, `ios.bundleIdentifier` в app.json.

- Rolling-summary внутри одной длинной сессии — транскрипт-персист и recap уже закрывают главный разрыв; ступенчатая свёртка — следующая итерация.
