# PabLito — как всё устроено (обзор после v2)

Свод того, как система работает целиком: модули, поток данных, ключевые механики. Для быстрого въезда в проект.

## Слои

```
UI (React Native)      ChatScreen · CityStrip · Pronunciation/LearnedWords/Memory Modal
        │
Оркестрация            src/hooks/usePablito.ts  — «мозг»: связывает всё, ведёт сессию
        │
Ядро (чистая логика)   mood.ts · memory-logic.ts · progress.ts · pronunciation.ts · personality.ts
        │
Данные / внешнее       memory.ts (SQLite) · api/navy.ts (LLM/STT/TTS) · api/weather.ts · voice.ts
```

## Поток одного хода разговора
1. Пользователь пишет/говорит → `send()` (голос: запись → `whisper-1` → текст).
2. Ход пишется в транскрипт (`messages` в SQLite) и в контекст модели.
3. `rebuildSystem` собирает системный промпт: персона + настроение (`moodToPrompt`) + педагогика + память (`getFactsForPrompt`, релевантные моменты, recap прошлой сессии) + режим дня + прогресс.
4. `chatStream` (api.navy, `deepseek-v4-flash`) стримит ответ; служебные блоки прячутся на лету.
5. Из ответа парсятся `[[MEM]]` (факты/слова/обещания/эмо-сигнал/момент) и `[[SET]]` (профиль).
6. Память обновляется (типизированные факты с заменой устаревших, SM-2 для слов), настроение — по событиям хода (сигнал модели + регексы-фолбэк + паузы).
7. Ответ озвучивается (TTS NavyAI → фолбэк `es-AR`).

## Ключевые механики
- **Настроение v2** (`mood.ts`): быстрые оси energy/warmth/patience (дрейф к baseline) + медленная ось **bond** (близость, копится неделями, пол 15, почти без дрейфа). Мягкое насыщение у краёв, потолок дельты за ход. «Тонус дня» (энергичный/спокойный/игривый/собранный/ностальгирующий) — раз в день. Дуга близости: знакомство→приятели→близкие→давние друзья открывает всё более тёплое поведение.
- **Память v2** (`memory.ts` + `memory-logic.ts`): факты с `kind`/`subject_key`/`importance`/`status`; новый факт в той же теме аннулирует старый. Имя и главная цель закреплены (importance=3). Дедуп по нормализованному тексту. Выборка в промпт — по бюджету символов. Эмоциональные моменты — отдельная таблица, всплывают к месту. Транскрипт персистится, сессия восстанавливается ≤30 мин по последней активности; при уходе в фон — авто-резюме.
- **SRS** (`memory-logic.ts`): SM-2-подобный ease-factor; «сказал верно вживую» двигает вперёд, трудные назад; новые слова дозируются.
- **Педагогика** (`personality.ts`): погружение i+1, мягкие исправления, режим Урок/Болталка, курс сценариев до жизни в БА.
- **Произношение** (`pronunciation.ts`): read-aloud, сравнение whisper-транскрипта с целью, пословный счёт.
- **Прогресс** (`progress.ts`): стрик (не падает в 0), вехи, счётчик до Аргентины.
- **Этика**: в персоне и движке — без вины за паузы, без дефицита/зависимости, не подменяет живое общение.

## Надёжность (после код-ревью)
Кэш промиса БД (гонка миграций), очередь записи, guard от двойной инициализации и двойного send, таймаут стриминга, защита от утечки служебных блоков в чат/TTS, приватная очистка памяти (включая транскрипт), валидация даты и флага онбординга, бюджет промпта. Детали и отложенное — в `CODE_REVIEW.md`.
