# APK прямо из GitHub → на макбук → на Samsung

Идея: APK собирает **сам GitHub** (бесплатно, GitHub Actions). Тебе не нужен Android-тулчейн на маке — только запушить код и нажать кнопку. Готовый файл скачаешь со страницы **Releases**.

---

## Шаг 1. Запушить проект в свой репозиторий (на макбуке)

Открой Терминал на маке:

```bash
cd ~/Desktop/Паблито
rm -rf .git                     # убрать служебную папку, оставшуюся от подготовки
git init
git add .
git commit -m "PabLito"
git branch -M main
git remote add origin https://github.com/StarDust1508/PabLito.git
git push -u origin main
```

> Если Git попросит логин — введи логин GitHub и **токен** вместо пароля (GitHub → Settings → Developer settings → Personal access tokens). Пароль GitHub в терминале давно не принимает.

Ключ NavyAI сюда НЕ коммитится (`.env` в `.gitignore`) — он добавляется отдельно секретом (шаг 2).

## Шаг 2. Добавить ключ как секрет репозитория

1. В браузере: репозиторий → **Settings** → **Secrets and variables** → **Actions**.
2. **New repository secret**.
3. Name: `NAVY_API_KEY`, Secret: твой **новый** ключ `sk-navy-…` (старый из чата перевыпусти).
4. Save.

## Шаг 3. Запустить сборку

1. Репозиторий → вкладка **Actions**.
2. Слева выбери **Build Android APK** → кнопка **Run workflow** → **Run workflow**.
3. Жди ~15–20 минут (первая сборка дольше). Зелёная галка = готово.

## Шаг 4. Скачать APK на макбук

- Репозиторий → раздел **Releases** (справа) → релиз **«PabLito — последний APK»** → скачай файл **`pablito.apk`**.
- (Или во вкладке Actions открой завершённый запуск → блок **Artifacts** → `pablito-apk`.)

## Шаг 5. Перекинуть на Samsung и поставить

- AirDrop с мака на Samsung не работает, поэтому отправь файл себе любым способом: Telegram «Избранное», Google Drive, почта или USB-кабель.
- На телефоне открой `pablito.apk` → при установке разреши «Установка из неизвестных источников».
- Готово — иконка Паблито на экране. При первом запуске он поздоровается первым, спросит имя и дату переезда.

## Обновить приложение позже
Запушил изменения → снова **Actions → Run workflow** → скачал новый `pablito.apk` → поставил поверх.

## Если сборка упала
Это бывает на первой сборке из-за мелочей окружения. Открой упавший запуск в **Actions**, скопируй последние строки лога с ошибкой и пришли мне — поправлю workflow. Частые причины и правки уже учтены (New Architecture выключен, лицензии SDK принимаются, релиз подписывается debug-ключом для установки «для себя»).

## Запасной путь — EAS (если у тебя есть аккаунт Expo)
```bash
npm install -g eas-cli
eas login
eas build -p android --profile preview   # соберёт APK в облаке Expo и даст ссылку
```
`eas.json` с профилем `preview` уже в проекте.
