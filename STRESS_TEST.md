# PabLito — инженерный стресс-тест

Честный разбор архитектуры: где сломается, что недодумано, что уже починено.
Помечено: 🔴 блокер · 🟠 высокий · 🟡 средний · ⚪ мелочь · ✅ исправлено в этом заходе.

## 0. Допущения, которые нельзя проверить

**🔴 API NavyAI не подтверждён по докам.** `api.navy/docs` закрыт проверкой Cloudflare на бота — обходить её нельзя, поэтому эндпоинты я взял по стандарту OpenAI, который NavyAI заявляет. Конкретно под вопросом:
- `/audio/speech` (TTS): имя модели `tts-1` и голоса `onyx/alloy` — это OpenAI-имена. У NavyAI они могут называться иначе или TTS может не быть вовсе. **Падение обрабатывается**: если TTS не отвечает — озвучка откатывается на системный голос `es-AR` (`expo-speech`), приложение не ломается.
- `/audio/transcriptions` (whisper): формат multipart и поле `text` в ответе — стандарт OpenAI.
- Стриминг ответов — не проверял, см. ниже.
- **Что делать:** зайти в доки из своего кабинета и сверить три вещи — базовый URL, точный путь и имя голоса для TTS, поддержку `stream:true`. Правится только в `src/api/navy.ts`.

**🟠 Ключ вшит в сборку.** `EXPO_PUBLIC_NAVY_API_KEY` попадает в JS-бандл и вытаскивается из APK декомпиляцией. Для приложения «только для себя» — терпимо. Полное решение — прокси (Cloudflare Worker), ключ живёт на сервере. Готов сделать отдельно.

## 1. Разговор и модель

**🟠✅ Неограниченный рост контекста — ИСПРАВЛЕНО.** Раньше в модель уходила вся история сессии — токены, стоимость и риск упереться в лимит росли без предела. Теперь отправляется системный промпт + последние 20 ходов (`windowed()` в `usePablito.ts`). Полная история хранится в памяти для будущего свёртывания.
- *Остаточный нюанс:* детали разговора старше 20 ходов Паблито «забудет» в рамках сессии, если они не попали в долгую память через блок `[[MEM]]`. Хорошее развитие — рллинг-суммаризация каждые N ходов.

**🟠 Нет стриминга ответов.** Ответ приходит целиком — для живого собеседника ощущается как задержка (особенно на `deepseek-v4-pro`). Плюс озвучка стартует только после полного ответа. Путь решения: SSE через `expo/fetch` (стриминговое тело) или `react-native-sse`, печатать по токенам.

**🟡 Нет таймаута/отмены у запроса чата.** При зависшей сети запрос висит, `busy` не снимается, ввод заблокирован. Нужен `AbortController` + таймаут ~30 c с понятной ошибкой.

**🟡 Протокол памяти `[[MEM]]` хрупок.** Модель может: не выдать блок, выдать битый JSON (тогда факт просто теряется — парсер это переживает), или в редком случае проговорить служебный текст. Регекс ловит блок в любом месте ответа и вырезает до показа/озвучки, так что в TTS он не попадёт. Улучшение — вынести извлечение фактов в отдельный дешёвый вызов `deepseek-v4-flash` в конце сессии, а не просить модель делать это в каждом ответе.

## 2. Голос (это языковое приложение — тут тонко)

**🟠 Whisper «причёсывает» грамматику.** Самый неочевидный нюанс. Whisper транскрибирует то, что ты *хотел* сказать, часто исправляя падежи/спряжения. Значит по голосу Паблито может **не увидеть твою реальную ошибку** — а мягкое исправление ошибок было твоим требованием. Ошибки сохраняются только в **текстовом** вводе.
- *Варианты:* (а) для тренировки грамматики — печатать; (б) добавить оценку произношения отдельным инструментом; (в) принять как лимит и корректировать по тексту. Нужно твоё решение.

**🟡 Нет обратной связи по произношению/акценту.** Приложение слышит слова, но не оценивает, как ты их произнёс. Для «носителя языка» это заметный пробел. Отдельная большая фича (форсированное выравнивание/фонемы).

**🟡 Запись без ограничений.** Нет максимальной длины, отмены и индикатора уровня. Долгая запись = большой аплоад и задержка STT. Добавить лимит ~60 c и кнопку отмены.

**⚪ Голос `es-AR` может отсутствовать на устройстве.** Тогда системная озвучка возьмёт дефолтный испанский (не аргентинский). На Samsung обычно ставится через «Настройки → Синтез речи».

**⚪ base64 всего mp3 в памяти.** Для длинных ответов TTS — заметная нагрузка. Для реплик обычной длины ок.

## 3. Память и прогресс

**🟠 История сессии не переживает перезапуск приложения.** `messages` и полная история — только в оперативной памяти. Убил приложение посреди разговора — контекст диалога потерян. **Долгие факты и словарь сохраняются** (SQLite), так что «кто ты» Паблито помнит; забывается только незаписанная ткань текущего разговора. Решение — сохранять транскрипт в БД и поднимать при старте.

**🟠✅ Ранние факты терялись — ИСПРАВЛЕНО.** `getFacts` брал только 30 свежих; имя (самый первый факт) со временем выпадало. Теперь возвращается объединение свежих + 5 самых первых фактов.

**🟡 Факты дедуплицируются только по точному совпадению.** «le gusta el fútbol» и «le gusta el futbol» осядут как два. Не критично; можно нормализовать.

**🟡 Разбор словаря по `" - "` хрупок.** Если в переводе есть « - », разобьётся неверно. Сменить разделитель на `||` в протоколе.

**⚪ `finishSession` не вызывается.** Таблица сессий пишет только старт. Пригодится, когда добавим свёртку сессии в конце.

## 4. Настроение (главная фича — проверяем честно)

**🟢 Логика рабочая, не мок.** Состояние в БД, правила обновления от событий, дрейф к базовому уровню, подмешивание в промпт. Значения зажаты в 0..100.

**🟡 Эвристики событий грубые (регексы).** `STRUGGLING/PLAYFUL/GOOD_EFFORT` определяются по ключевым словам — бывают ложные срабатывания (например, `che` внутри слова). Это лишь дополнение к модели, катастрофы нет. Точнее — попросить саму модель возвращать сигнал настроения в `[[MEM]]`.

**⚪ Возможное «залипание».** При однотипных сессиях оси могут надолго уходить в край; дрейф в начале каждой сессии это лечит.

## 5. Города и погода (новое)

**🟢 Open-Meteo без ключа**, мульти-город одним запросом. Время — из `utc_offset_seconds`, тикает локально; при неудаче обновления показываются прошлые данные с меткой «оффлайн».

**🟡 Первый экран без сети** покажет «—:—» и плейсхолдеры, пока не пройдёт запрос. Кэш в SQLite между запусками — простое улучшение.

**⚪ Сдвиг пояса фиксируется на момент запроса.** Переход на летнее/зимнее время между обновлениями (раз в 15 мин) в теории даст минутную неточность — практически незаметно.

## 6. Сборка и версии

**🟡 `expo-av` объявлен устаревшим** в новых SDK (курс на `expo-audio`/`expo-video`). На SDK 52 работает; при апгрейде SDK — миграция записи/плеера.

**🟡 APK собирается только на твоей стороне** (EAS/Android Studio) — в песочнице нет Android-тулчейна. Команды в README. Проверь `npm run typecheck` после `npm install` — автономные модули я уже прогнал компилятором, остальное требует полного набора Expo-типов.

**⚪ Тестов нет.** Для ядра (mood/memory/weather — чистые функции) юнит-тесты добавляются легко и окупаются.

## 7. Приватность

Аудио и текст уходят в NavyAI; факты о тебе лежат в SQLite открытым текстом на устройстве. Для личного приложения нормально. Если параноидально — шифровать БД (SQLCipher) и/или поднять прокси.
